# Cloud Forensics Project

This repository contains resources and tools for cloud forensics across multiple cloud providers, including AWS, Azure, and GCP. Each folder is dedicated to a specific cloud provider and contains relevant files and configurations for managing and investigating cloud environments.

## Project Structure

### `AWS/`
This folder contains resources and configurations for AWS cloud forensics.
- **`cloudformation-webapp.yaml`**: A CloudFormation template to deploy a web application infrastructure in AWS.
- **`README.md`**: Documentation with AWS CLI commands for managing and investigating AWS resources, such as EC2 instances, Auto Scaling Groups, and EBS volumes.

### `Azure/`
This folder is reserved for Azure-specific forensic tools and configurations. (Currently empty or under development.)

### `GCP/`
This folder is reserved for GCP-specific forensic tools and configurations. (Currently empty or under development.)

## Usage
- Navigate to the folder corresponding to the cloud provider you are working with.
- Follow the instructions or use the provided scripts and templates to manage and investigate cloud resources.

## Contribution
Feel free to contribute by adding new tools, templates, or documentation for cloud forensics in any of the supported cloud providers.
